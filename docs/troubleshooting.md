# Troubleshooting

Use this guide in **Symptom → Cause → Fix** format. Every item includes exact verification steps.

## 1) Blank then sync on focus

**Symptom**
- Dashboard appears empty/older momentarily when tab gains focus, then repopulates.

**Likely cause**
- Focus refresh sequence hydrates local cache first, then pulls newer cloud state after debounce.
- Another device updated cloud data, causing local replacement.

**Fix**
1. Wait for focus refresh cycle to finish.
2. Check status bar and toast for “Updated from another device”.
3. If local unsynced edits were expected, import latest backup with merge.
4. Re-test with only one active editing session.

**Verification steps**
1. Open app in two browsers/devices.
2. On device A, make change and wait for sync complete.
3. On device B (stale tab), focus window.
4. Confirm data updates to device A version and status remains healthy.

## 2) “Auth state failed” errors

**Symptom**
- Status message contains `Auth state failed: ...`.

**Likely cause**
- Session callback failure, invalid redirect configuration, or blocked browser storage.

**Fix**
1. Verify Supabase redirect allowlist includes current site URL.
2. Ensure browser is not blocking required storage/cookies.
3. Sign out and sign back in.
4. Retry in a clean browser profile to isolate extension interference.

**Verification steps**
1. Reload app.
2. Perform sign-in.
3. Confirm signed-in display appears and action cards are visible.
4. Confirm no `Auth state failed` status appears after reload.

## 3) “Import failed: invalid backup format”

**Symptom**
- Import modal fails with `Import failed: invalid backup format.`

**Likely cause**
- Uploaded JSON is not the expected wrapper or is corrupted.

**Fix**
1. Ensure JSON parses correctly.
2. Ensure top-level object contains `state` key.
3. Prefer backups generated by app Export.
4. Re-run import with valid file.

**Verification steps**
1. Export a fresh backup from app.
2. Import exported file in merge mode.
3. Confirm success status and visible data unchanged.

## 4) Missing data after import (cloud overwritten)

**Symptom**
- Expected records not present after import/sync.

**Likely cause**
- Overwrite mode replaced local snapshot.
- Cloud conflict resolution pulled newer cloud state over local imported state.

**Fix**
1. Determine whether import used merge or overwrite.
2. Immediately export current state for safe checkpoint.
3. Re-import correct backup (usually merge first).
4. If needed, temporarily isolate to one device/session during recovery.

**Verification steps**
1. Compare pre/post import item counts.
2. Add a test action and confirm numbering progression.
3. Confirm sync status reaches synced without conflict warning.

## 5) Rate limits / SMTP issues (generic)

**Symptom**
- Sign-in or email-based auth may fail intermittently with provider throttling messages.

**Likely cause**
- Authentication provider throttling, SMTP delivery delay, or temporary service limits.

**Fix**
1. Retry after cooldown period.
2. Reduce repeated rapid auth attempts.
3. Validate provider health/status dashboard.
4. Confirm sender/domain settings are correct in auth provider.

**Verification steps**
1. Wait recommended cooldown.
2. Retry single sign-in attempt.
3. Confirm session is established and persisted on reload.

## 6) Favicon caching issues

**Symptom**
- Browser tab shows old/missing favicon after deployment.

**Likely cause**
- Aggressive browser cache of `favicon.svg`.

**Fix**
1. Hard refresh page.
2. Clear site data/cache for app origin.
3. Reopen tab or browser.

**Verification steps**
1. Open site in private/incognito window.
2. Confirm favicon displays expected icon.
3. Return to normal window and confirm after cache clear.

## 7) Filter parsing issues across newlines

**Symptom**
- `@person` or `#tag` expected matches do not appear in filter lists.

**Likely cause**
- Token not matching allowed pattern boundaries or characters.
- Rich text introduced punctuation/formatting that split token.

**Fix**
1. Ensure token format is contiguous: `@Name` / `#Topic` with `[A-Za-z0-9_-]` characters.
2. Avoid spaces inside token.
3. Save item and re-open filter dropdown.
4. Reset filters to `All` if stale selection persists.

**Verification steps**
1. Create action with multiline content containing valid token.
2. Save and confirm token appears in dropdown.
3. Select token filter and verify expected rows are shown.
